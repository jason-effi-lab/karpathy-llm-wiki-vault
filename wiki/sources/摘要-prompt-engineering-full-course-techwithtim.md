## 核心摘要

Tech With Tim 出品的提示工程完整视频课程（2026-03-11），系统讲解从基础到进阶的 13 个提示技术。核心观点：提示工程本质是"用自然语言编程"，通过**引导（Steering）而非命令（Commanding）**来精确控制模型输出。现代推理模型已内置 CoT，显式链式思维在顶级模型上的重要性下降，但结构化输出、少样本示例、约束指令仍是高价值技术。

## 主要内容章节

| 章节 | 核心观点 |
|------|----------|
| 什么是提示工程 | 用自然语言取代代码指令模型行为 |
| 如何更快提示 | 使用语音输入（Whisper Flow）代替打字 |
| LLM 的"思考"方式 | 本质是 token 预测；记忆依赖上下文注入 |
| 引导 vs 命令 | 指定长度/格式/重点 > 简单命令 |
| 明确具体 & 设定场景 | 角色 + 受众 + 语气 + 格式 = 更佳输出 |
| 少样本提示 | 给出输入/输出示例对，引导模式识别 |
| 思维链 | 要求分步推理；现代模型已自动执行 |
| 结构化输出 | 指定 JSON/表格/Markdown 格式并给出示例 |
| 约束与否定指令 | 告诉模型"不要做什么"往往比"要做什么"更有效 |
| 迭代细化 | 多轮对话中逐步修正提示 |
| 面试式提示 | 让模型先提问再作答，获取缺失上下文 |
| 高级技术与参数 | Temperature、系统提示、多模态 |
| 常见错误 | 过度模糊、忽略格式、不给示例 |

## 关键洞见

- **提示长度悖论**：详细提示通常更好，但冗余内容适得其反
- **否定指令效果**：研究人员常用大量"不要做"清单提升效果
- **上下文注入**：99% 情况下模型看到的不只是你的提示，还有系统注入的历史对话
- **语音提示**：用语音输入比打字更快，且自然产生更详细的提示
